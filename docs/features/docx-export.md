# DOCX 公文导出功能

> 状态：v1 | 最后更新：2026-04-03
> 唯一真值（SSOT）：本文档是 DOCX 导出所有样式规范的权威来源。

## 功能概述

将当前打开的 Markdown 文件**一键导出**为符合 GB/T 9704-2012《党政机关公文格式》的 Word 文档（.docx）。

- 零外部依赖（不需要安装 Pandoc）
- 纯机械化转换（不需要大模型参与）
- 样式表干净（只含必要样式，无冗余英文样式）

---

## 技术方案

| 组件 | 选型 | 说明 |
|------|------|------|
| DOCX 生成 | `docx` npm v9.6.1 | 纯 JS OOXML 生成，2M+ 周下载 |
| Markdown 解析 | `unified` + `remark-parse` + `remark-gfm` + `remark-frontmatter` | 复用项目已有解析器 |
| 转换方式 | mdast AST → docx Document | 确定性映射，无 AI |

**转换流水线**：
```
Markdown 文本 → remark.parse() → mdast AST → ast-to-docx 纯函数 → docx.Document → Packer.toBuffer() → .docx 文件
```

| 每页行数 | 22 行 | |

### 行距与段落

| 参数 | 值 | docx 单位 |
|------|-----|-----------|
| 行距 | 固定值 28pt | 560 twip |
| 段前 | 0 | |
| 段后 | 0 | |

### 标题样式

GB/T 9704 第 7.3.3 条：各级标题首行缩进 2 字符（左空二字），回行顶格。

| 级别 | Markdown | Word 样式 | 中文字体 | 英文字体 | 字号 | 加粗 | 对齐 |
|------|----------|-----------|---------|---------|------|------|------|
| 公文标题 | `# H1` | Title | 方正小标宋简体 | Times New Roman | 二号 22pt | 否 | 居中，无缩进 |
| 一级标题 | `## H2` | 标题 1 | 黑体 SimHei | Arial | 三号 16pt | 否 | 左+缩进2字符 |
| 二级标题 | `### H3` | 标题 2 | 楷体 KaiTi | Arial | 三号 16pt | 否 | 左+缩进2字符 |
| 三级标题 | `#### H4` | 标题 3 | 仿宋 FangSong | Times New Roman | 三号 16pt | 是 | 左+缩进2字符 |
| 四级标题 | `##### H5` | 标题 4 | 仿宋 FangSong | Times New Roman | 三号 16pt | 否 | 左+缩进2字符 |
| 五~七级 | `###### H6` 等 | 标题 5-7 | 仿宋 FangSong | Times New Roman | 三号 16pt | 否 | 左+缩进2字符 |

> **关键技术细节**：使用 `styles.default.headingN` API 覆盖 Word 内建样式，保留 `w:name="heading N"`。
> 如果用 `paragraphStyles` + 自定义 `name`，Word 会创建新样式导致重复。

### 标题编号规范

| 级别 | 编号格式 | 示例 |
|------|---------|------|
| 一级 | 一、二、三、… | 一、总体要求 |
| 二级 | （一）（二）（三）… | （一）指导思想 |
| 三级 | 1. 2. 3. … | 1. 明确目标 |
| 四级 | （1）（2）（3）… | （1）具体措施 |

> 标题编号是 Markdown 文本内容的一部分，不由 Word 样式自动生成。

### 强调样式（Markdown `**加粗**`）

公文中不使用"加粗"来表示强调，而是通过**切换字体**来区分。

| 参数 | 值 |
|------|-----|
| 中文字体 | 楷体 KaiTi |
| 英文字体 | Arial |
| 字号 | 三号 16pt |
| 加粗 | 否 |

> Markdown 的 `**文字**` 导出为楷体（不加粗），实现"用字体区分强调"的公文排版理念。
> 楷体与黑体同属无衬线组，搭配 Arial。

## 字体策略

### 公文五大字体及英文配对

每种中文字体搭配固定的英文字体，在代码中以拼音 PascalCase 命名：

| 变量名 | 中文字体 | 英文字体 | 分组 | 用途 |
|--------|---------|---------|------|------|
| `XiaoBiaoSong` | 方正小标宋简体 | Times New Roman | 衬线 | 公文标题 |
| `HeiTi` | 黑体 SimHei | Arial | 无衬线 | 一级标题、表头 |
| `KaiTi` | 楷体 KaiTi | Arial | 无衬线 | 二级标题、强调 |
| `FangSong` | 仿宋 FangSong | Times New Roman | 衬线 | 正文、三/四/五~七级标题 |
| `SongTi` | 宋体 SimSun | Times New Roman | 衬线 | 页码 |
| `CodeFont` | 宋体 SimSun | Consolas | 等宽 | 代码块 |

> **配对规则**：黑体和楷体属无衬线组，配 Arial；仿宋和宋体属衬线组，配 Times New Roman。

### 不使用 GB2312 编码

所有字体使用 Windows/macOS 系统自带的现代字体名，不使用 `_GB2312` 后缀。

### OOXML rFonts 四槽位

Word 通过字符 Unicode 范围自动选择对应字体槽位：

- `eastAsia`：CJK 中文字符
- `ascii`：基本拉丁字符（英文、数字、Em Dash）
- `hAnsi`：扩展拉丁字符

中英混排时只需声明一次，Word 自动切换。页码中全部是英文字符（Em Dash + 数字），实际使用的是 ascii 槽位的 Times New Roman。

### 跨平台兼容

| 角色 | Windows | macOS（Word 自动回退） |
|------|---------|----------------------|
| 公文标题 | FZXiaoBiaoSong-B05S | 需安装（字体文件在 `fonts/FZXBSJW.TTF`） |
| 一级标题 | SimHei | Heiti SC |
| 二级标题 | KaiTi | Kaiti SC |
| 正文 | FangSong | STFangsong |
| 页码 | SimSun | Songti SC |

## UX 交互设计

### 编辑器按钮

- 编辑器右上角自定义 Word 文档 SVG 图标（dark/light 双主题）
- 仅在 Markdown 文件中显示

### 两种导出命令

| 命令 | 入口 | 行为 |
|------|------|------|
| `导出公文 DOCX` | 按钮 + 命令面板 | 弹出保存对话框 |
| `快速导出公文 DOCX` | 仅命令面板 | 直接保存到同目录 |

### 交互流程

1. 未保存修改检测 → 保存并导出 / 直接导出 / 取消
2. 带进度条导出（`withProgress` + 可取消）
3. 成功通知（含文件大小）+ "打开文件" / "在文件管理器中显示"
4. 上次导出路径记忆（workspaceState）
5. 失败时"查看日志"按钮 → Output Channel "DOCX 导出"

## 边缘情况处理

| 场景 | 处理 |
|------|------|
| 空文档 | 有效空 DOCX（一个空段落） |
| YAML Frontmatter | 自动跳过，不输出到 DOCX |
| Mermaid 代码块 | 等宽代码块 + 提示文字 |
| LaTeX 数学 | 等宽纯文本公式 |
| 远程图片（http） | 黑色占位文字 + URL |
| 缺失本地图片 | 黑色占位文字 + 路径 + 汇总警告 |
| 超宽图片 | 自动等比缩放到版心宽度 156mm |
| 链接 | 输出为黑色普通文字（公文无蓝色超链接） |
| 所有文字颜色 | 统一黑色（#000000） |

## 样式干净度

使用 `styles.default.headingN` 覆盖 Word 内建样式，不创建额外自定义段落样式。
最终 DOCX 样式面板仅显示 Word 内建样式（标题、标题 1-7、正文）+ 一个 Strong 字符样式。
